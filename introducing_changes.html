Introducing changes (2016.11)

Introduction

Implementing a feature is just one step of a much longer process. This page should serve as a high-level introduction to the various phases an idea can go through before it becomes a feature that provides benefit to players and the company.

Phases of a Change

Diagram Version

Table Version

Step Actions Keep in mind See also
1. Identify possible problem See gap in the KPIs, OR get player feedback, OR have a hunch, OR see something cool in another game, etc. There are many ways of gathering potential problems, and pretty much all of them are valid. There are an infinite number of problems that we can come up with. That's why step 2 is so important.
2. Decide whether to proceed 2.1 What are some possible solutions? This step isn't about finding the solution, but more about exploring the possibilities and having a rough indication if we can feasibly solve the problem.
2.2 How valuable would a possible solution be? / Why should we solve this? Find a way to measure the impact that the solution might have. The impact should be on something corresponding to the general goals for the product - things like retention or monetization. This is a very important step.
  • Why prepare targets / measure impact for changes?
  • Preparing early expectations about Revenue Potential
2.3 Prioritize it against other changes Compare to other changes, and do the most valuable change first. This is where the change gets added to the roadmap.
3. Figure out how to solve the problem 3.1 Conception and design phase It is important to make sure the problem we're trying to solve stays clear here, and decisions are centered around solving that problem.
3.2 Grooming / implementation evaluation Discussion about the technical feasibility.
4. Attempt to solve the problem 4.1 Development
4.2 Release
5. Learn from the attempt Check back on the results after release.

Why:

  • To determine whether the original problem was solved so it can inform our future steps on this problem
  • So that we and others can learn for other changes
  • Knowing that this step will come also helps keep us focused earlier in the process
